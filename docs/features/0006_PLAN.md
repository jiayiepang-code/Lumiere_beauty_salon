## Implementation Details

### Phase 1: Backend - `admin/customers/list.php`

**1.1 Session Initialization & Security**

```
- Start session with secure configuration (from auth_check.php pattern)
- Check admin authentication (requireAdminAuth())
- Generate CSRF token for forms
- Set page title and base path
```

**1.2 POST Handler for Form Submissions**

```
- Check request method (POST for edit/delete operations)
- Validate CSRF token
- Route to appropriate handler based on action parameter
- Set $_SESSION['status'] with result
- Display header/footer
```

**1.3 DELETE Handler (Within POST)**

When action === 'delete':

- Validate customer_email parameter
- Check for foreign key constraints:
  - Count bookings referencing this customer_email in `booking` table
  - If bookings exist, set error status
- Execute DELETE prepared statement:
  ```sql
  DELETE FROM customer WHERE customer_email = ?
  ```
- Set session status: 'Customer deleted successfully' or error message
- Set `$_SESSION['deleted_email']` to hide row on client-side

**1.4 UPDATE Handler (Within POST)**

When action === 'edit':

- Validate customer_email parameter
- Validate input fields (first_name, last_name, phone)
- Build dynamic UPDATE query based on provided fields
- Handle password reset separately:
  - If password field provided, hash with `password_hash($password, PASSWORD_BCRYPT)`
  - Include in UPDATE query
- Execute prepared statement with bound parameters
- Set session status: 'Customer updated successfully' or validation errors
- Set `$_SESSION['updated_email']` to update row on client-side

## Security Requirements

### All CRUD Operations Must Use Prepared Statements

**SELECT Example (list.php)**:

```php
$sql = "SELECT customer_email, first_name, last_name, phone, created_at FROM customer ORDER BY last_name ASC";
$stmt = $conn->prepare($sql);
$stmt->execute();
$result = $stmt->get_result();
```

**UPDATE Example (update.php)**:

```php
$sql = "UPDATE customer SET first_name = ?, last_name = ?, phone = ? WHERE customer_email = ?";
$stmt = $conn->prepare($sql);
$stmt->bind_param("ssss", $first_name, $last_name, $phone, $customer_email);
$stmt->execute();
```

**DELETE Example (delete.php)**:

```php
$sql = "DELETE FROM customer WHERE customer_email = ?";
$stmt = $conn->prepare($sql);
$stmt->bind_param("s", $customer_email);
$stmt->execute();
```

### Input Validation

- **Email**: Already validated as primary key format
- **Phone**: Strip special characters, validate Malaysian format (starts with 60 or 01)
- **Names**: Trim whitespace, no special characters beyond apostrophes/hyphens
- **Password**: Minimum 8 characters, hash with `password_hash($password, PASSWORD_BCRYPT)`

### CSRF Protection

- All forms include CSRF token from `getCSRFToken()`
- Validate token on server before processing any state-changing operation
- Reference: `api/admin/staff/update.php` for CSRF validation pattern

### Output Escaping

- Escape all customer data when displaying in HTML: `htmlspecialchars($data, ENT_QUOTES, 'UTF-8')`
- Use `escapeHtml()` function in JavaScript for DOM manipulation

```javascript
/**
 * Format currency amount in Malaysian Ringgit
 */
function formatCurrency(amount) {
  return `RM ${parseFloat(amount || 0).toFixed(2)}`;
}

/**
 * Format date as relative time (e.g., "2 days ago")
 */
function formatRelativeTime(dateString) {
  if (!dateString) return "Never";

  const date = new Date(dateString);
  const now = new Date();
  const diffMs = now - date;
  const diffDays = Math.floor(diffMs / (1000 * 60 * 60 * 24));

  if (diffDays > 90) return `${Math.floor(diffDays / 30)} months ago`;
  if (diffDays > 30) return `${Math.floor(diffDays / 30)} month ago`;
  if (diffDays > 0) return `${diffDays} day${diffDays > 1 ? "s" : ""} ago`;

  const diffHours = Math.floor(diffMs / (1000 * 60 * 60));
  if (diffHours > 0) return `${diffHours} hour${diffHours > 1 ? "s" : ""} ago`;

  return "Today";
}

/**
 * Get customer status based on last visit
 */
function getCustomerStatus(lastVisit, totalBookings, createdAt) {
  if (!lastVisit) return "new";

  const daysSinceVisit = Math.floor(
    (new Date() - new Date(lastVisit)) / (1000 * 60 * 60 * 24)
  );
  const daysSinceRegistration = Math.floor(
    (new Date() - new Date(createdAt)) / (1000 * 60 * 60 * 24)
  );

  if (daysSinceRegistration < 30 && totalBookings < 3) return "new";
  if (daysSinceVisit <= 30) return "active";
  if (daysSinceVisit <= 90) return "inactive";
  return "at-risk";
}

/**
 * Render status badge HTML
 */
function renderStatusBadge(status) {
  const badges = {
    active:
      '<span class="status-badge active"><span class="status-dot"></span>Active</span>',
    inactive:
      '<span class="status-badge inactive"><span class="status-dot"></span>Inactive</span>',
    "at-risk":
      '<span class="status-badge at-risk"><span class="status-dot"></span>At Risk</span>',
    new: '<span class="status-badge new"><span class="status-dot"></span>New</span>',
  };
  return badges[status] || badges["new"];
}

/**
 * Render tier badge based on total bookings
 */
function renderTierBadge(totalBookings) {
  if (totalBookings >= 20)
    return '<span class="badge badge-warning">VIP</span>';
  if (totalBookings >= 10)
    return '<span class="badge badge-info">Regular</span>';
  return '<span class="badge badge-secondary">New</span>';
}
```

### 2. Customer Segmentation Filters

**Objective:** Add dropdown filters to quickly segment customers by tier and status for targeted management.

**Features:**

- **Tier Filter:** All | VIP (20+) | Regular (10-19) | New (0-9)
- **Status Filter:** All | Active | Inactive | At Risk | New
- **Spending Filter:** All | High Value ($500+) | Medium ($100-$499) | Low (<$100)
- Filters work in combination (e.g., "VIP + At Risk" = high-value customers who need re-engagement)

**UI Location:**

- Add filter dropdowns below search bar, above table
- Use same styling as sort dropdown for consistency
- Show active filter count: "2 filters applied"

**Implementation:**

```javascript
// Add to filterCustomers() function
function applySegmentationFilters(customers) {
  const tierFilter = document.getElementById("tierFilter").value;
  const statusFilter = document.getElementById("statusFilter").value;
  const spendingFilter = document.getElementById("spendingFilter").value;

  return customers.filter((customer) => {
    // Tier logic
    if (tierFilter !== "all") {
      const tier =
        customer.total_bookings >= 20
          ? "vip"
          : customer.total_bookings >= 10
          ? "regular"
          : "new";
      if (tier !== tierFilter) return false;
    }

    // Status logic
    if (statusFilter !== "all") {
      const status = getCustomerStatus(
        customer.last_visit,
        customer.total_bookings,
        customer.created_at
      );
      if (status !== statusFilter) return false;
    }

    // Spending logic
    if (spendingFilter !== "all") {
      const spent = parseFloat(customer.total_spent || 0);
      if (spendingFilter === "high" && spent < 500) return false;
      if (spendingFilter === "medium" && (spent < 100 || spent >= 500))
        return false;
      if (spendingFilter === "low" && spent >= 100) return false;
    }

    return true;
  });
}
```

**Business Value:**

- Identify at-risk VIP customers for retention campaigns
- Find new customers for welcome promotions
- Target inactive regulars for re-engagement offers
- Discover high-spend potential (few bookings but high average spend)
